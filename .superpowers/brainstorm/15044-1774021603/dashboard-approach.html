Progression Dashboard — Design Approach

The current dashboard is functional but basic. How ambitious should the redesign be?

A

Enhanced Current

Keep same structure. Add stat cards, sport filter, rolling averages on scatter plots, fix time filter bugs. ~30 min work.

Pros

  • Quick, minimal risk
  • Familiar layout

Cons

  • Still noisy scatter plots
  • No FTP/NP tracking
  • No YoY visual comparison
B

Performance-First Redesign

Restructure around "Am I getting better?" — stat cards, progression charts with rolling averages + scatter, FTP progression, NP tracking, ride-type-split monthly trends, visual YoY comparison, improved fitness history.

Pros

  • Answers the key question clearly
  • Consistent with Overview quality
  • FTP trend = most valuable cycling metric
  • Rolling averages make trends readable

Cons

  • More panels to build
  • FTP computation is a heavier query
C

Deep Analytics

Everything in B plus: best effort curves over time (5min/20min power), training polarization (zone distribution trends monthly), power duration curve overlay (this month vs last), personal records timeline with milestones.

Pros

  • Most comprehensive analysis
  • Coaching-grade insights

Cons

  • Heavy stream queries may be slow
  • Some charts only useful with 6+ months data
  • Risk of dashboard overload

My recommendation: B — it delivers the most value without overloading. We can always add C's analytics later as a separate "Training Analysis" dashboard.
